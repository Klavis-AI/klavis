# Klavis Sandbox (Sandbox-as-a-Service)

Klavis Sandbox gives you **ephemeral, fully controllable** application environments (e.g., Gmail, Snowflake) to safely test and evaluate agents without touching production data.

## Lifecycle

<Steps>
  <Step title="Create">
    Request a sandbox and get an MCP server URL for that isolated instance.
  </Step>
  <Step title="Initialize (seed)">
    Load a deterministic “world state” (records, messages, files, etc.) via the Sandbox API.
  </Step>
  <Step title="Interact (MCP)">
    Let your agent use MCP tools against the sandbox as if it were the real app.
  </Step>
  <Step title="Dump (verify)">
    Snapshot the full sandbox state to programmatically validate outcomes.
  </Step>
  <Step title="Reset / Delete">
    Wipe the sandbox back to a clean slate for the next test run.
  </Step>
</Steps>
